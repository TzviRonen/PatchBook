## What is PatchBook?

PatchBook publishes deep technical analyses of Windows kernel security patches. Each post is generated by [kernel-cve-pipeline](https://github.com/tzvironen/kernel-cve-pipeline) — an automated system that:

1. Downloads the patched and unpatched versions of affected Windows binaries from Microsoft's Winbindex
2. Runs **Ghidriff** to produce function-level binary diffs
3. Uses a Claude-powered patch identifier to pinpoint the exact changed function and classify the vulnerability
4. Generates a structured post covering the root cause, patch mechanics, and exploitation primitive

## What you'll find here

Each post covers:

- **The bug** — root cause analysis with decompiled pseudo-C, often including race windows, uninitialized memory, or missing bounds checks
- **The patch** — what Microsoft changed and why it closes the vulnerability
- **Exploitation primitive** — how an attacker could turn the bug into a privilege escalation, information leak, or remote code execution

## Audience

Posts assume familiarity with C, Windows kernel internals (object manager, memory manager, IRP dispatch), and common vulnerability classes (TOCTOU, UAF, pool overflows).
